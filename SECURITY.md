# Security Policy

## Supported Versions

We actively release security updates for the following versions of **Zlija**:

| Version | Supported          |
| ------- | ------------------ |
| 0.1.x   | :white_check_mark: |

## Reporting a Vulnerability

If you discover a security vulnerability within **Zlija** (such as data corruption bugs, memory leaks, or denial of service issues), please **do not** open a public GitHub issue.

Instead, report the issue privately by emailing: **your_email@example.com**

### What to Include in Your Report:
* A detailed description of the vulnerability.
* Steps to reproduce the issue (a minimal Rust example `main.rs` is highly appreciated).
* Potential impact of the vulnerability.

We will acknowledge receipt of your report within **48 hours** and keep you updated on the fix progress before making a public disclosure.
